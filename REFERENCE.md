## 一、README.md 中文翻譯

# 模型火箭飛行控制電腦

#### 以伺服機為基礎的飛行控制系統，為模型火箭提供主動穩定性與資料記錄功能。

## 專案範疇

本專案旨在為業餘愛好者、學術研究者及模型火箭團隊提供一套易於理解且功能豐富的開源飛行電腦設計方案，採用伺服機驅動的主動穩定控制與資料記錄功能。本專案提供方塊圖、電路圖、原始碼，以及設計背後的工程思路，供所有對模型火箭感興趣的人參考。

電子零件採用市面上容易取得的消費性產品，並為業餘火箭套件選配了 3D 列印改裝件以簡化組裝流程。此設計具有模組化特性，方便自定義修改，亦可適用於可操控鴨翼、尾翼或推力向量控制（但後兩種尚未實際測試）。

目前實作的飛行穩定機制是透過「向重力反方向偏轉」來讓火箭保持垂直飛行，不支援任何形式的目標導引。

為了讓本專案保持實作性與迭代性，採用了一些非慣例的設計決策：
- 使用靜態建模而非完整的飛行動力學模擬，設計保守的轉向方案；
- 使用非高G設計的感測器融合輸出，預期在高加速度下需要替換；
- 使用電木板與 CNC 銑削 PCB 原型，雖然體積比工廠製 PCB 大，但可快速測試與修改。

作者計畫持續分析飛行記錄，以描述不同機身的轉向特性，並逐步引入不同的狀態空間模型以提升轉向精度。

## 版本規劃

專案分為多個版本，每個版本在不同火箭機身上增加更多功能。目前 V1 和 V2 已完成，並記錄了多次發射的飛行資料以分析主動飛行控制行為：

- **V1** ✓（開源、已完成）：功能完整的飛行電腦，搭載 BNO055 IMU、主動垂直飛行導向翼控制，以及 SD 卡資料記錄，使用 BT-60 Estes Green Eggs 機身。相關工程內容已發表於 [IEEE UEMCON 2025 學術論文](https://ieeexplore.ieee.org/document/11267717)。
- **V2** ⧗（開源、已完成，準備 GitHub 文件中）：新增氣壓計、電池充電器、GPS、SDIO SD 記錄、LoRa、地面站及中斷驅動轉向。採用全新 3 英寸 PCB 和 Apogee Mako 火箭設計，已完成飛行測試。
- **V3** ☐（未決定）：66mm 火箭機身，搭配伺服控制尾翼以蒐集更多資料。
- **V4** ☐（閉源）：飛抵目標位置及/或沿預定軌跡飛行功能，模擬高功率火箭離開大氣層並進入軌道的真實需求。

後續進階版本的飛行電腦軟體將為私有閉源。若希望合作開發更進階的合法火箭設計或客製化電腦，歡迎聯絡作者。

## V1 設計概述

V1 的第一個里程碑是完成垂直主動飛行導向，使火箭持續向地球反方向飛行。這需要製作飛行電腦、飛行電腦軟體，以及模型火箭硬體。

### V1 飛行電腦

在設計火箭之前，選定 RP2040 作為飛行電腦並採用 Pi Pico 實作。選擇此微控制器的主要考量為：快速時脈、多核心處理及 PIO 埠。

微控制器先在麵包板上與各種感測器、SD 卡及伺服機進行初步接線測試。軟體進展到足夠的概念驗證後，才開始火箭組裝，飛行電腦從麵包板移至焊接電木板與 3D 列印的解決方案。由於飛行電腦工作在 3.3V，使用簡單的被動式 SD 卡轉 MicroSD 轉接器，在其上直接焊接跳線，讓飛行電腦直接連接至用於記錄資料的 MicroSD 卡。

### PIO 的應用

RP2040 的 PIO 埠在伺服控制上特別有用，因為伺服機需要約 1ms 到 2ms 的精確脈衝來設定位置。利用 PIO，建立了 4 個狀態機，其計時完全獨立於核心程序，無需使用中斷，確保翼片能以一致的伺服脈衝計時保持精確的設定位置，不受主程式狀態影響。

翼片位置變更是透過將新的脈衝計時延遲傳送至對應伺服機的 PIO 暫存器來實現。轉向變更確實需要核心程式運算資料並傳送更新，因此需要處理器非常快速地循環執行程式以提供平滑轉向。

V2 的程式修改將 IMU 輪詢和伺服設定改為中斷驅動程式流程，無論處理器的迴圈速度如何，都能持續平滑地主動轉向。

為了初步了解 PIO 實作及其組合語言程式碼，首先為黃色狀態 LED 和紅色錄製 LED 撰寫了 PIO 狀態機程式進行主動 LED 控制。LED 的閃爍速度讓火箭操作員了解火箭的目前狀態及發射準備情況。

### V1 火箭設計

V1 火箭基於改裝的 Estes Green Eggs 模型火箭套件建構。使用 OpenRocket 軟體建模火箭的簡單飛行特性，包括重心（CoG）和壓力中心（CoP），以及使用不同 Estes 引擎的預期飛行動力學模擬。

從 rocketryforum.com 下載了由用戶 BEC 創建的原版 Green Eggs 火箭套件 OpenRocket 存檔，與實體 V1 火箭進行比較。進行了微小調整以匹配 V1 的構建，然後將飛行控制、感測器和主動轉向的附加模組建模到設計中。

### V1 伺服外殼與轉向翼

建模飛行特性並決定伺服機和轉向翼位置後，在 Fusion360 中為伺服外殼創建了 3D 模型。採用直接將飛行控制翼安裝到伺服機上的設計，無需任何額外連桿，以節省空間並保持簡潔。

伺服外殼位於引擎和降落傘之間，必須承受來自引擎的降落傘彈射藥力，並讓此力通過，彈出火箭鼻錐並展開降落傘，同時不損壞伺服機或外殼。外殼以 PLA 材料進行 3D 列印，並使用環氧樹脂永久固定在火箭機身管內。

轉向翼在 Fusion360 中建模，以匹配用於飛行建模的 OpenRocket 設計。金屬線被黏合到伺服連接器凸塊的中心以提供支撐，而不只是摩擦裝配。這些半剛性金屬線貫穿整個翼片，在翼片頂部彎折固定。

### V1 感測器圓盤

設計了一個圓盤形感測器外殼（以 Fusion360），用於裝載 BNO055 IMU 和 PA1616D GPS 模組。感測器模組旋入 3D 列印的 PLA 外殼，整個組件滑入裝有飛行電腦的透明綠色管中。圓盤的形狀確保 IMU 感測器的正確方向，並避免感測器與火箭的錯位。

外殼還安裝了兩個開關：一個控制飛行電腦電源開/關，另一個控制 SD 卡資料記錄。關閉並重新開啟資料記錄會開始新的記錄文件。

### V1 飛行電腦軟體

軟體分為兩部分：PIO 狀態機和主 C 程式。狀態機和 C 程式相互獨立運行，通過暫存器互動，改變 PIO 狀態機的行為。

**V1 中 PIO 狀態機控制：**
- LED 使用延遲數字控制閃爍
- 伺服脈衝使用延遲數字（以 125MHz 時脈計算）

**BNO055 IMU 設定：**  
IMU 設定為 NDOF 模式，使用加速計、陀螺儀和磁力計的感測器融合通過卡爾曼濾波器輸出非常穩定的資料。此過濾資料提供每個軸的重力向量，用於轉向控制：
- 理想垂直狀態：Z 軸重力向量 = 9.8，X 和 Y 軸 ≈ 0
- 火箭偏斜時：X 和 Y 的變化直接用於控制相關轉向翼的伺服脈衝

**V1 限制與計畫改進：**
- SD 卡目前以 10Hz 存儲；更頻繁的存儲導致轉向抖動（SPI 速度限制）
- GPS UART 解析速度慢導致轉向抖動，目前停用
- BMP390 氣壓計初始化問題，V1 未啟用
- 計畫：將 IMU 輪詢和轉向移至中斷驅動；SPI 換成 SDIO（透過 PIO）

**使用的函式庫：**  
[carlk3/no-OS-FatFS-SD-SDIO-SPI-RPi-Pico](https://github.com/carlk3/no-OS-FatFS-SD-SDIO-SPI-RPi-Pico) 用於 SPI 和後續 SDIO 的 SD 卡介面。

## 蒐集的飛行資料

截至撰寫時，V1 已記錄 9 次飛行，V2 已記錄 3 次飛行（加上一次未啟用電子設備的飛行）。

透過 MATLAB 處理加速度、重力和方向資料，再積分成速度和位置資料，部分飛行已成功映射為飛行路徑。一次飛行路徑的模擬對比顯示，降落傘彈射藥在預期的近頂點處觸發，真實數據與模擬投影相當接近。隨著 V2 開始加入氣壓計數據，飛行映射和描述變得更加容易。

## 二、教程規劃

本教程面向具備基礎電子和程式設計能力的業餘愛好者，分為五個模組循序漸進學習。

### 模組一：飛行電腦基礎概念（2-3 小時）

**學習目標：** 理解飛行控制系統的組成與工作原理

**內容：**
1. 主動穩定 vs 被動穩定的差異
2. 飛行電腦的功能架構：感測 → 計算 → 執行
3. 重心（CoG）與壓力中心（CoP）的關係
4. 伺服機脈衝寬度調變（PWM）原理
5. OpenRocket 軟體入門：火箭建模與模擬

**實作練習：**
- 在 OpenRocket 中建立簡單火箭模型
- 觀察 CoG/CoP 比例對穩定性的影響
- 計算在不同引擎下的預期飛行高度

---

### 模組二：RP2040 與 PIO 程式設計（4-6 小時）

**學習目標：** 掌握 RP2040 的 PIO 功能用於精確時序控制

**內容：**
1. Raspberry Pi Pico / RP2040 架構介紹
2. PIO（可程式 I/O）概念與組合語言
3. 伺服機 PIO 狀態機撰寫
4. LED 狀態指示 PIO 程式
5. 主程式與 PIO 透過暫存器的互動

**實作練習：**
```c
// 伺服脈衝 PIO 範例概念
// 1ms = 125,000 時脈週期（125MHz）
// 2ms = 250,000 時脈週期
// 對應伺服行程範圍：1ms ~ 2ms
```
- 建立測試電路：RP2040 + 1 個伺服機
- 撰寫 PIO 程式控制伺服角度
- 測試多個伺服機同步控制（4 個 PIO 狀態機）

---

### 模組三：感測器整合與資料融合（4-6 小時）

**學習目標：** 讀取 IMU 資料並實作基本轉向邏輯

**內容：**
1. BNO055 IMU 介紹：I2C 通訊協議
2. NDOF 模式與卡爾曼濾波器概念
3. 重力向量的意義與轉向應用
4. 讀取加速度向量、陀螺儀、方向角
5. MicroSD 卡 SPI 介面資料記錄

**實作練習：**
- 接線：RP2040 + BNO055（I2C）
- 讀取並列印重力向量（gx, gy, gz）
- 實作轉向邏輯：`servo_position = clamp(gy * gain, min, max)`
- 將感測器資料以 CSV 格式寫入 SD 卡
- 離線分析 SD 卡資料（Python / MATLAB）

---

### 模組四：硬體組裝與 3D 列印（6-10 小時）

**學習目標：** 完成飛行電腦硬體組裝與機身改裝

**內容：**
1. 電木板焊接技巧（代替麵包板）
2. Fusion360 基礎：設計伺服外殼
3. PLA 3D 列印參數建議（填充率、外殼厚度）
4. 火箭機身改裝：切割開孔、安裝伺服機
5. 接線整理與斷路保護設計

**組裝步驟：**
1. 設計並列印感測器圓盤（sensor puck）
2. 設計並列印伺服外殼
3. 焊接電木板：RP2040 + BNO055 + SD 卡模組 + 伺服機
4. 製作 4 路伺服快拆連接器
5. 組裝轉向翼（加金屬線強化）

---

### 模組五：飛行測試與資料分析（持續進行）

**學習目標：** 發射、記錄並分析真實飛行資料

**內容：**
1. 發射前檢查清單
2. 飛行電腦狀態指示：LED 燈號含義
3. SD 卡資料讀取與 Python 資料分析
4. 積分加速度資料還原飛行路徑
5. 調整轉向增益（gain）改善穩定性

**資料分析範例（Python）：**
```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

df = pd.read_csv('flight_log.csv')
# 積分加速度 -> 速度 -> 位置
dt = df['time'].diff().fillna(0)
vz = np.cumsum(df['az'] * dt)
z = np.cumsum(vz * dt)
plt.plot(df['time'], z)
plt.title('飛行高度（重建）')
plt.show()
```

## 四、水火箭可行性探討

### 4.1 水火箭基本原理

水火箭（Water Rocket）利用**壓縮空氣 + 水**作為推進劑。原理：
1. 向寶特瓶（機身）充入高壓空氣（一般 6-8 bar）
2. 壓力釋放時，水從噴嘴高速噴出（牛頓第三定律）
3. 水噴完後，剩餘壓縮空氣繼續提供少量推力

**典型性能參數：**
- 推力持續時間：0.1 ~ 0.5 秒（極短）
- 峰值加速度：10 ~ 30g（視水量與壓力）
- 典型飛行高度：30 ~ 150m
- 機身直徑：82mm（標準 1.5L）~ 110mm（2L 寶特瓶）

### 4.2 與模型火箭的關鍵差異

| 特性 | 模型火箭（Estes） | 水火箭 |
|------|-------------------|--------|
| 推進劑 | 固體燃料火藥 | 壓縮空氣 + 水 |
| 推力持續時間 | 0.5 ~ 3 秒 | 0.1 ~ 0.5 秒 |
| 峰值加速度 | 5 ~ 30g | 10 ~ 50g |
| 機身直徑 | 29 ~ 54mm（BT 管） | 82 ~ 110mm（寶特瓶） |
| 可重複使用 | 否（引擎一次性） | 是 |
| 法規限制 | 視重量/衝量，需申報 | 相對寬鬆 |
| 降落傘展開 | 引擎彈射藥觸發 | 需自製機構 |
| 每次費用 | 引擎費用約 100-500 元 | 幾乎零耗材 |

### 4.3 技術可行性分析

#### 可直接移植的部分

| 功能 | 可行性 | 說明 |
|------|--------|------|
| RP2040 控制器 | ✅ 完全可行 | 輕量、3.3V 低功耗 |
| BNO055 IMU 資料記錄 | ✅ 完全可行 | 飛行時間短但資料仍有價值 |
| PIO 伺服機控制 | ✅ 完全可行 | 同樣的脈衝寬度邏輯 |
| SD 卡資料記錄 | ✅ 完全可行 | 分析水火箭飛行特性 |
| GPS 位置記錄 | ✅ 完全可行 | 追蹤落點 |
| BMP390 氣壓計 | ✅ 完全可行 | 高度測量更重要（無法靠引擎計算） |
| LoRa 遙測 | ✅ 完全可行 | 即時傳輸資料到地面站 |

#### 需要調整的部分

**1. BNO055 高G表現問題（重要）**

水火箭初始加速度可達 20-50g，遠高於模型火箭（Estes 引擎通常 7-15g）。BNO055 的 NDOF 感測器融合模式在高加速度下穩定性下降，原始 README 也提到此問題。

**建議方案：**
- 短時間推力階段（< 0.5s）：接受感測器失準，待加速度降低後恢復姿態估算
- 使用純陀螺儀積分（不依賴重力向量）作為推力期備援
- 考慮換用高G IMU：ADXL377（±200g 加速計）或 ICM-42688-P

**2. 降落傘展開機構**

水火箭沒有引擎彈射藥，需自製展開機構：

| 方案 | 原理 | 難度 |
|------|------|------|
| 氣壓觸發 | 利用機身內剩餘壓力在頂點彈出鼻錐 | 簡單，但不易精確控制 |
| 伺服機觸發 | 飛行電腦偵測頂點後用伺服機鬆開鼻錐 | 中等，需頂點偵測演算法 |
| 彈簧鎖定 + 電磁閥 | 磁閥通電釋放鼻錐 | 需要額外功耗 |

**建議：** 使用氣壓計偵測頂點，搭配伺服機觸發降落傘（最可靠）。

**3. 機身介面設計**

寶特瓶機身（82-110mm）與標準火箭身管（29-54mm）尺寸差異大，需客製化：
- 重新設計伺服外殼適配寶特瓶直徑
- 鼻錐需 3D 列印（市面無現成品）
- 考慮多段組合的機身（充氣段 + 電子艙）

**4. 防水設計**

水火箭發射時水霧噴濺，電子元件需適當保護：
- 使用防水外殼或保形塗料（conformal coating）
- RP2040 和主板的接頭使用矽膠密封
- MicroSD 卡槽加防塵蓋

**5. 重量控制**

水火箭對重量更敏感，電子系統應盡量輕量化：
- Pi Pico：21g
- BNO055 模組：~3g
- MicroSD：~2g
- 電池（500mAh LiPo）：~12g
- 伺服機（4 x MG90S）：4 x 13.4g = ~54g
- **電子系統總重：約 100g**

若要搭載主動控制翼，電子系統總重約 100g，對 1.5L 寶特瓶火箭（總重約 300-500g）是可接受範圍，但會顯著降低飛行高度。

### 4.4 推薦的水火箭飛行電腦架構（簡化版）

若以水火箭為目標，可採用此精簡架構：

```
[RP2040 Pi Pico]
    ├── I2C → BNO055 IMU（姿態）
    ├── I2C → BMP390 氣壓計（高度 + 頂點偵測）
    ├── SPI → MicroSD（10Hz 資料記錄）
    ├── PIO × 2 → 伺服機（2 軸轉向翼）
    ├── PIO × 1 → 降落傘伺服機（頂點釋放）
    ├── GPIO → 狀態 LED
    └── UART → LoRa 模組（可選，遙測用）
```

**簡化理由：**
- 水火箭通常只需 2 軸控制（減少伺服機至 2 個降低重量）
- 氣壓計在無引擎的水火箭上比模型火箭更關鍵（頂點偵測）
- GPS 可選配（依需求）

### 4.5 可行性結論

**結論：水火箭上使用此飛行控制系統是可行的，但需要針對性調整。**

| 面向 | 評估 |
|------|------|
| 電子系統移植 | 高度可行（核心架構相同） |
| 機身適配 | 需客製化（尺寸不同） |
| 高G加速度挑戰 | 需注意感測器融合失準，可軟體補償 |
| 降落傘展開 | 需新增機構（氣壓計頂點偵測 + 伺服觸發） |
| 防水需求 | 需適當保護 |
| 成本效益 | 極高（水火箭無引擎成本，可多次測試） |
| 學習價值 | 高（相同原理，更安全的測試環境） |

**建議學習路徑：**
1. 先以水火箭進行電子系統測試（低成本、安全、可重複）
2. 驗證 IMU 姿態感測、SD 記錄、伺服控制
3. 實作頂點偵測與降落傘展開
4. 待系統穩定後升級至固體燃料模型火箭

水火箭是**理想的低成本迭代平台**，特別適合學習和驗證飛行電腦各子系統，而不必每次測試都消耗昂貴的引擎。
